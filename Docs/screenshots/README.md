# Screenshot checklist

No screenshots are included in the initial cleaned public release because no
existing App Store or simulator captures have yet been reviewed for repository
publication. This avoids replacing real product evidence with synthetic images
or exposing private test data.

Before adding screenshots:

1. Use a clean simulator or a dedicated non-personal test device.
2. Use obviously fictional conversations and invalid provider placeholders.
3. Hide API keys, custom headers, provider balances, account IDs, email
   addresses, notification previews, and local file names.
4. Review the status bar, photo picker, document picker, sidebar, and keyboard
   suggestions for personal information.
5. Remove image metadata before committing.
6. Prefer captures that match the public source snapshot and label any older
   App Store version accurately.

Suggested coverage:

- conversation and streaming response;
- provider/model configuration with secret fields empty;
- conversation search and branching;
- knowledge-base management using fictional documents;
- iPad layout, if available.
